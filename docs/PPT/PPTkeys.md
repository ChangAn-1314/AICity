# 国赛级别项目PPT制作要点与技巧

## 一、PPT内容结构（"危机方案我做成"框架）

### 1. 危 - 痛点与需求分析（1-2页）
- **客户痛点**：用户遇到的问题、纠结或抱怨
- **市场需求**：国家需求、社会需求、个人需求
- 用数据和案例说明问题的严重性

### 2. 机 - 政策背景与资源分析（2-3页）
- **政策支持**：国家/地方政策（如二十大、教育部文件等）
- **市场规模**：用数据展示市场刚需
- **资源整合**：人、财、物、技术优势

### 3. 方 - 产品/服务与技术方案（核心，5-8页）
- **产品介绍**：新技术、新服务、新方案
- **技术架构**：系统架构图、技术栈
- **创新点**：算法、模型、独特功能（**必须突出智能化**）
- **功能演示**：思维导图 + 效果截图/视频

### 4. 案 - 典型案例与成效（1-2页）
- 实际应用案例
- 与企业/机构的合作成果
- 可用短视频替代，**让用户说比自己说效果好**

### 5. 我 - 团队介绍（2-3页）
- **负责人单独一页**：突出核心能力和项目贡献
- **团队成员**：专业互补、分工明确
- **导师/顾问**：作用要具体，不能只是挂名

### 6. 做 - 发展规划（1-2页）
- 业务规划：未来3-5年计划
- 技术迭代路线
- 融资/合作规划（如适用）

### 7. 成 - 项目成效与价值（1页）
- 带动就业、社会效益
- 学科交叉、产学研成果

### 8. 结束语（1页）
- 项目凝练升华的一句话
- **掷地有声**，不要用"谢谢评委"之类的弱结尾
- 例："挖掘XX金矿，推动科技XX，助力XX振兴！"

---

## 二、PPT制作技术规范

| 项目 | 要求 |
|------|------|
| **页数** | 20页左右，最多不超过25页 |
| **字号** | 30号以上，关键数字加粗变色 |
| **配色** | 每页不超过3种颜色，对比度要高 |
| **动画** | 尽量少用，网评时评委不会播放 |
| **排版** | 一页一个核心信息，简洁清晰 |
| **图表** | 用图表替代文字，越直观越好 |

## 三、答辩演示技巧

### 说辞准备
- **提前写好完整说辞**，反复熟读，确保不卡壳
- 比赛时可把说辞放在屏幕旁辅助（评委看不到）
- 时间控制在规定范围内（通常10分钟）

### 演示准备
- **提前在本地启动项目**，测试所有功能
- 检查网络和电脑电量
- 准备备用方案（录屏视频）

### 应答技巧
1. **不熟悉的问题**：说出知道的部分，承认不足，表示会继续学习
2. **完全不知道**：引导到你熟悉的功能上
3. **熟悉的问题**：充分展开，把优点都说出来

## 四、核心加分项

### 1. 项目命名
- **好听、好记、易定位**
- 格式：XX者（领航者、开创者）、国内/全球首个XX

### 2. 技术亮点
- **必须结合算法体现智能化**（推荐算法、深度学习、NLP等）
- 设计大赛不看技术深度，看能否解决问题

### 3. 项目选题
- 结合国家战略（智慧城市、乡村振兴、碳中和等）
- 解决实际问题，有社会价值
- 与所在地区相关（更容易过省赛）

### 4. 答辩顺序
- **尽量争取中间位置答辩**
- 开头评委会压分，后期评委疲劳
- 中间位置最容易获得高分

---

## 五、针对AI智慧城市项目的特别建议

1. **可视化效果**：大屏、3D地图、数据可视化是加分项
2. **实时数据**：展示系统处理实时数据的能力
3. **多端协同**：Web + 移动端 + 大屏的联动
4. **AI能力**：突出大模型、预测分析、智能决策等功能
5. **落地价值**：强调对城市管理的实际帮助

---

## 六、常见评委问题预判

1. 项目的核心算法是什么？如何实现智能化？
2. 与现有产品/系统相比，你们的创新点在哪？
3. 项目的数据来源是什么？如何保证数据安全？
4. 团队分工是怎样的？每个人负责什么？
5. 项目的商业化/落地计划是什么？
6. 技术难点是什么？如何解决的？
7. 项目是否已经实际应用？效果如何？

## 七、PPT视觉设计建议

### 配色方案
- **科技感**：深蓝 + 青色 + 白色
- **智慧城市**：深色背景 + 霓虹色点缀
- 与项目UI风格保持一致

### 图表类型
- 架构图：展示系统层次
- 流程图：展示业务逻辑
- 数据图：柱状图、饼图展示成效
- 地图：展示应用区域

### 封面要素
- 项目名称
- 参赛赛道/组别
- 学校名称
- 团队名称
- 指导老师
- 项目Logo
